# MindCampus 项目结构分析报告

## 项目概述

**项目名称**: MindCampus（高校学生心理健康智能干预平台）  
**版本**: v3.9.0  
**技术框架**: 基于若依（RuoYi）框架 - SpringBoot + Vue 前后端分离架构  
**开发语言**: Java 17 + Vue 3  
**项目描述**: 专业的大学生心理健康服务平台

---

## 二、项目整体结构特点分析

### 2.1 架构设计特点

#### **（1）前后端分离架构**
- **后端**: 基于 SpringBoot 3.5.4，采用 RESTful API 设计
- **前端**: 
  - Web 端：Vue 3.5.16 + Element Plus 2.10.7 + Vite 6.3.5
  - 移动端：uni-app（Vue 2）多端适配（H5、小程序、App）
- **通信方式**: HTTP/HTTPS + JSON，JWT Token 认证

#### **（2）多模块化设计（Maven 多模块工程）**
项目采用典型的若依框架分层架构，包含8个核心模块：

| 模块名称 | 模块职责 | 依赖关系 |
|---------|---------|---------|
| **mc-admin** | 应用启动模块，Web服务入口 | 依赖所有其他模块 |
| **mc-framework** | 框架核心模块，提供安全、配置、切面等 | 依赖 mc-system |
| **mc-common** | 通用工具模块，提供工具类、注解、异常 | 无依赖（基础模块）|
| **mc-system** | 系统管理模块，RBAC权限管理 | 依赖 mc-common |
| **mc-project** | 核心业务模块，心理健康平台业务 | 依赖 mc-common, mc-system |
| **mc-ai** | AI智能模块，阿里云百炼集成 | 依赖 mc-framework |
| **mc-quartz** | 定时任务模块，Quartz调度 | 依赖 mc-common |
| **mc-generator** | 代码生成器模块，Velocity模板 | 依赖 mc-common |

**模块依赖层级**:
```
mc-admin (应用层)
    ├── mc-framework (框架层)
    │       └── mc-system (系统层)
    │               └── mc-common (工具层)
    ├── mc-project (业务层)
    ├── mc-ai (AI层)
    ├── mc-quartz (任务层)
    └── mc-generator (工具层)
```

---

### 2.2 若依框架核心组件应用

#### **（1）权限管理体系（RBAC）**
- **用户-角色-菜单-部门** 四维权限模型
- **数据权限**: 通过 `@DataScope` 注解实现行级权限控制
- **核心实体**:
  - `SysUser`: 用户信息
  - `SysRole`: 角色管理
  - `SysMenu`: 菜单/按钮权限
  - `SysDept`: 部门组织架构

#### **（2）安全认证机制**
- **Spring Security + JWT**: 无状态认证
- **核心组件**:
  - `JwtAuthenticationTokenFilter`: JWT令牌验证过滤器
  - `TokenService`: Token生成与解析服务
  - `SecurityConfig`: 安全配置（白名单、加密策略）
  - `UserDetailsServiceImpl`: 用户详情加载

#### **（3）数据访问层技术栈**
- **MyBatis Plus 3.5.10**: 增强型ORM框架
- **多数据源支持**: 
  - `DynamicDataSource`: 动态数据源切换
  - `@DataSource` 注解实现数据源路由
- **Druid 1.2.23**: 数据库连接池与监控

#### **（4）系统监控与日志**
- **操作日志**: `@Log` 注解 + `LogAspect` 切面记录
- **登录日志**: `SysLogininfor` 记录登录行为
- **服务监控**: 
  - CPU、内存、JVM监控（`Server`相关类）
  - Druid SQL监控
  - 在线用户监控

#### **（5）代码生成器**
- **模板引擎**: Apache Velocity 2.3
- **生成内容**:
  - Java后端：Controller、Service、Mapper、Domain
  - 前端：Vue页面、API接口
  - SQL脚本、MyBatis XML映射
- **配置化**: 通过 `gen_table` 和 `gen_table_column` 表配置生成规则

## 四、总结

**MindCampus** 是一个基于若依（RuoYi）框架开发的 **高校学生心理健康智能干预平台**，具有以下核心特征：

1. **架构成熟**: 采用 SpringBoot + Vue 前后端分离，模块化设计清晰
2. **权限完善**: 完整的RBAC权限体系，支持数据权限
3. **功能全面**: 继承若依框架的系统管理、监控、日志、代码生成等功能
4. **业务聚焦**: 针对心理健康领域，提供学生管理、咨询师管理、问卷调查、评估分析
5. **AI赋能**: 集成阿里云百炼大模型，实现智能评估与分析
6. **多端覆盖**: Web管理端 + uni-app移动端，满足不同场景需求
7. **开发高效**: 代码生成器、完善的工具类库、统一的开发规范

该项目是若依框架在 **教育心理健康领域** 的典型应用案例，既保留了框架的核心优势，又针对业务需求进行了合理扩展。
