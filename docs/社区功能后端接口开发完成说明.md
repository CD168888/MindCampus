# 社区功能后端接口开发完成说明

## 问题描述

小程序请求 `/app/community/post/list` 接口时报错：
```
No static resource app/community/post/list
```

原因：后端只有管理后台的 Controller (`/community/post`)，缺少小程序专用的 App Controller (`/app/community/post`)。

## 解决方案

### 1. 创建 App 端 Controller

#### AppCommunityPostController.java
**路径：** `mc-project/src/main/java/com/mc/community/controller/AppCommunityPostController.java`

**功能：**
- `GET /app/community/post/list` - 查询帖子列表（分页）
- `GET /app/community/post/{postId}` - 查询帖子详情
- `POST /app/community/post` - 发布帖子
- `POST /app/community/post/{postId}/like` - 点赞帖子
- `POST /app/community/post/{postId}/unlike` - 取消点赞帖子

#### AppCommunityCommentController.java
**路径：** `mc-project/src/main/java/com/mc/community/controller/AppCommunityCommentController.java`

**功能：**
- `GET /app/community/post/{postId}/comments` - 查询帖子的评论列表
- `POST /app/community/comment` - 发表评论
- `POST /app/community/comment/{commentId}/like` - 点赞评论
- `POST /app/community/comment/{commentId}/unlike` - 取消点赞评论

### 2. 更新实体类

#### CommunityPost.java
添加了以下字段：
- `userId` - 用户ID
- `userName` - 用户名
- `userAvatar` - 用户头像
- `title` - 帖子标题
- `images` - 图片列表（逗号分隔）
- `likeCount` - 点赞数
- `commentCount` - 评论数
- `viewCount` - 浏览数
- `isLiked` - 是否已点赞（非数据库字段）

### 3. 更新 Service 层

#### ICommunityPostService.java
添加了以下方法：
- `incrementViewCount(Long postId)` - 增加浏览量
- `incrementCommentCount(Long postId)` - 增加评论数
- `likePost(Long postId, Long userId)` - 点赞帖子
- `unlikePost(Long postId, Long userId)` - 取消点赞帖子

#### ICommunityCommentService.java
添加了以下方法：
- `likeComment(Long commentId, Long userId)` - 点赞评论
- `unlikeComment(Long commentId, Long userId)` - 取消点赞评论

### 4. 更新 Mapper 层

#### CommunityPostMapper.java
添加了以下方法：
- `incrementViewCount(Long postId)` - 增加浏览量
- `incrementCommentCount(Long postId)` - 增加评论数

#### CommunityPostMapper.xml
- 更新了 resultMap，添加新字段映射
- 更新了 selectCommunityPostVo，查询新字段
- 更新了 insert 和 update 语句，支持新字段
- 添加了 incrementViewCount 和 incrementCommentCount 的 SQL

### 5. 数据库更新脚本

**文件：** `sql/update_community_post_table.sql`

**执行步骤：**
```sql
-- 1. 添加新字段
ALTER TABLE `community_post`
ADD COLUMN `user_id` bigint(20) NULL COMMENT '用户ID' AFTER `post_id`,
ADD COLUMN `user_name` varchar(100) NULL COMMENT '用户名' AFTER `user_id`,
ADD COLUMN `user_avatar` varchar(500) NULL COMMENT '用户头像URL' AFTER `user_name`,
ADD COLUMN `title` varchar(200) NULL COMMENT '帖子标题' AFTER `user_avatar`,
ADD COLUMN `images` text NULL COMMENT '图片列表（逗号分隔）' AFTER `content`,
ADD COLUMN `like_count` bigint(20) DEFAULT 0 COMMENT '点赞数' AFTER `images`,
ADD COLUMN `comment_count` bigint(20) DEFAULT 0 COMMENT '评论数' AFTER `like_count`,
ADD COLUMN `view_count` bigint(20) DEFAULT 0 COMMENT '浏览数' AFTER `comment_count`;

-- 2. 添加索引
CREATE INDEX `idx_user_id` ON `community_post`(`user_id`);
CREATE INDEX `idx_create_time` ON `community_post`(`create_time`);
CREATE INDEX `idx_status` ON `community_post`(`status`);
```

## 注意事项

### 1. 点赞功能简化实现

当前点赞功能是简化实现，直接修改点赞数。实际项目中应该：
- 创建点赞记录表 (`community_like`)
- 记录用户点赞关系
- 防止重复点赞
- 支持查询用户是否已点赞

### 2. 用户信息获取

发布帖子和评论时，用户信息从 `SecurityUtils` 获取：
```java
Long userId = SecurityUtils.getUserId();
String userName = SecurityUtils.getUsername();
```

确保用户已登录并有有效的 token。

### 3. 图片上传

图片上传使用 `/common/upload` 接口，返回图片 URL。
多张图片用逗号分隔存储在 `images` 字段中。

### 4. 评论树结构

评论支持两级结构：
- 一级评论：`parentId` 为 null
- 二级评论（回复）：`parentId` 为父评论 ID

前端需要自行构建树形结构。

### 5. 权限控制

所有接口都需要登录态（token），通过 Spring Security 验证。
