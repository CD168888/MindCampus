# 问卷发送重复数据问题修复说明

## 问题描述

在问卷管理界面点击"发送问卷"后，测评管理界面会一次性添加两条或多条相同的数据记录。

## 问题原因

1. **缺少重复检查**：`sendQuestionnaire` 方法在插入数据前没有检查是否已存在相同的学生-问卷记录
2. **数据库缺少唯一约束**：`evaluation_result` 表没有对 `(student_id, questionnaire_id)` 设置唯一索引
3. **重复点击**：用户多次点击发送按钮会导致重复插入

## 修复方案

### 1. 后端代码修复

#### 1.1 Service层添加重复检查

**文件**：`mc-project/src/main/java/com/mc/questionnaire/service/impl/QuestionnaireServiceImpl.java`

**修改内容**：
- 在插入数据前检查是否已存在该学生的该问卷测评记录
- 如果已存在则跳过插入，返回 `false`
- 如果成功插入则返回 `true`
- 添加默认状态值（未完成、未读、未分析）

```java
@Override
public boolean sendQuestionnaire(Long questionnaireId, Long studentId) {
    // 检查是否已存在该学生的该问卷测评记录
    EvaluationResult query = new EvaluationResult();
    query.setStudentId(studentId);
    query.setQuestionnaireId(questionnaireId);
    List<EvaluationResult> existingResults = evaluationResultMapper.selectEvaluationResultList(query);
    
    // 如果已存在记录，则不重复插入
    if (existingResults != null && !existingResults.isEmpty()) {
        return false; // 返回 false 表示跳过
    }
    
    // 插入新的测评记录
    evaluationResultMapper.insertEvaluationResult(new EvaluationResult() {{
        setQuestionnaireId(questionnaireId);
        setStudentId(studentId);
        setCompletionStatus("0"); // 0-未完成
        setReadStatus("0"); // 0-未读
        setAiStatus("0"); // 0-未分析
    }});
    
    return true; // 返回 true 表示发送成功
}
```

#### 1.2 Controller层添加统计信息

**文件**：`mc-project/src/main/java/com/mc/questionnaire/controller/QuestionnaireController.java`

**修改内容**：
- 统计发送总数、成功数、跳过数、无学生信息数
- 返回详细的发送结果给前端

```java
@PostMapping("/send")
public R<java.util.Map<String, Object>> send(
        @RequestParam(value = "questionnaireId") Long questionnaireId,
        @RequestParam(value = "deptId") Long deptId) {
    
    List<SysUser> sysUsers = sysUserService.selectUserListByDeptId(deptId);
    
    // 统计发送结果
    int totalUsers = sysUsers.size();
    int sentCount = 0;
    int skippedCount = 0;
    int noStudentCount = 0;
    
    for (SysUser sysUser : sysUsers) {
        Student student = new Student();
        student.setUserId(sysUser.getUserId());
        List<Student> students = studentInfoService.selectStudentInfoList(student);
        
        if (!students.isEmpty()) {
            Student studentInfo = students.get(0);
            boolean isSent = questionnaireService.sendQuestionnaire(
                questionnaireId, studentInfo.getStudentId());
            if (isSent) {
                sentCount++;
            } else {
                skippedCount++; // 已存在，跳过
            }
        } else {
            noStudentCount++; // 用户未关联学生信息
        }
    }
    
    // 构建返回数据
    java.util.Map<String, Object> result = new java.util.HashMap<>();
    result.put("totalUsers", totalUsers);
    result.put("sentCount", sentCount);
    result.put("skippedCount", skippedCount);
    result.put("noStudentCount", noStudentCount);
    result.put("message", String.format(
        "发送完成：成功 %d 人，跳过 %d 人（已存在记录），无学生信息 %d 人", 
        sentCount, skippedCount, noStudentCount));
    
    return R.ok(result);
}
```

### 2. 数据库修复

#### 2.1 添加唯一索引

**文件**：`sql/fix_evaluation_result_duplicate.sql`

**内容**：
1. 清理已存在的重复数据（保留最早的记录）
2. 添加唯一索引 `uk_student_questionnaire (student_id, questionnaire_id)`

```sql
-- 1. 先删除可能存在的重复数据（保留最早的记录）
DELETE t1 FROM evaluation_result t1
INNER JOIN evaluation_result t2 
WHERE 
    t1.student_id = t2.student_id 
    AND t1.questionnaire_id = t2.questionnaire_id
    AND t1.result_id > t2.result_id;

-- 2. 添加唯一索引约束
ALTER TABLE evaluation_result 
ADD UNIQUE INDEX uk_student_questionnaire (student_id, questionnaire_id) 
COMMENT '学生问卷唯一索引：同一学生不能对同一问卷创建多条测评记录';
```

**执行方式**：
```bash
mysql -u root -p mindcampus < sql/fix_evaluation_result_duplicate.sql
```

### 3. 前端优化

#### 3.1 显示详细发送结果

**文件**：`mc-ui/MindCampus-Web/src/views/questionnaire/questionnaireinfo/index.vue`

**修改内容**：
- 解析后端返回的统计数据
- 显示友好的发送结果提示

```javascript
function confirmSend() {
  if (!selectedDept.value) {
    proxy.$modal.msgWarning("请选择要发送的部门")
    return
  }

  const loading = proxy.$loading({
    lock: true,
    text: '正在发送问卷...',
    spinner: 'el-icon-loading',
    background: 'rgba(0, 0, 0, 0.7)'
  })

  sendQuestionnaire(sendForm.value.questionnaireId, selectedDept.value.id)
    .then(response => {
      loading.close()
      
      // 显示详细的发送结果
      const data = response.data || {}
      
      if (data.sentCount !== undefined) {
        const { totalUsers, sentCount, skippedCount, noStudentCount } = data
        
        // 根据不同情况显示不同的提示
        if (sentCount === 0 && skippedCount === totalUsers) {
          // 全部跳过（已存在）
          proxy.$modal.msgWarning(`该部门的 ${totalUsers} 名学生已全部发送过此问卷，无需重复发送`)
        } else if (sentCount === totalUsers) {
          // 全部成功
          proxy.$modal.msgSuccess(`问卷发送成功！已向 ${selectedDept.value.label} 的 ${sentCount} 名学生发送问卷`)
        } else if (sentCount > 0) {
          // 部分成功
          let message = `发送完成！成功 ${sentCount} 人`
          if (skippedCount > 0) message += `，${skippedCount} 人已存在`
          if (noStudentCount > 0) message += `，${noStudentCount} 人无学生信息`
          proxy.$modal.msgSuccess(message)
        } else {
          // 一个都没成功
          let message = '未发送任何问卷'
          if (skippedCount > 0) message += `，${skippedCount} 人已存在记录`
          if (noStudentCount > 0) message += `，${noStudentCount} 人无学生信息`
          proxy.$modal.msgWarning(message)
        }
      } else {
        proxy.$modal.msgSuccess(data.message || '发送成功')
      }
      
      sendOpen.value = false
    })
    .catch(error => {
      loading.close()
      console.error('发送问卷失败:', error)
      proxy.$modal.msgError('发送失败，请重试')
    })
}
```

## 修复效果

### 修复前
- ❌ 多次点击发送按钮会创建多条重复记录
- ❌ 同一学生可以有多条相同问卷的测评记录
- ❌ 用户不知道发送结果详情
- ❌ 提示信息包含HTML标签，不够友好

### 修复后
- ✅ 同一学生同一问卷只能有一条测评记录
- ✅ 重复发送会自动跳过已存在的记录
- ✅ 显示详细的发送统计信息（成功、跳过、失败）
- ✅ 数据库层面通过唯一索引强制约束
- ✅ 使用简洁友好的纯文本提示
