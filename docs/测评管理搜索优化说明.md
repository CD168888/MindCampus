# 测评管理搜索优化说明

## 📝 优化背景

原测评管理界面的搜索功能使用**学生ID**和**问卷ID**进行查询，但实际使用中，用户更常用**学生姓名**和**问卷标题**进行搜索。由于 `evaluation_result` 表中没有存储这些字段，需要通过关联查询来实现。

## ✅ 优化内容

### 1. 数据库层优化

**问题**：`evaluation_result` 表只存储了 `student_id` 和 `questionnaire_id`，没有存储姓名和标题。

**解决方案**：使用 LEFT JOIN 关联查询 `student_info` 和 `questionnaire` 表。

#### 修改的 Mapper XML

**文件**：`mc-project/src/main/resources/mapper/evaluation/EvaluationResultMapper.xml`

**关键改动**：

1. **更新 ResultMap**，添加姓名和标题字段映射：
```xml
<resultMap type="EvaluationResult" id="EvaluationResultResult">
    <!-- ... 其他字段 ... -->
    <result property="studentName"    column="student_name"    />
    <result property="questionnaireTitle"    column="questionnaire_title"    />
</resultMap>
```

2. **修改查询SQL**，添加 LEFT JOIN：
```xml
<sql id="selectEvaluationResultVo">
    select 
        er.result_id, 
        er.student_id, 
        er.questionnaire_id, 
        er.total_score, 
        er.risk_level, 
        er.ai_analysis, 
        er.ai_status, 
        er.read_status, 
        er.completion_status, 
        er.create_by, 
        er.create_time, 
        er.update_by, 
        er.update_time, 
        er.remark,
        s.name as student_name,
        q.title as questionnaire_title
    from evaluation_result er
    left join student_info s on er.student_id = s.student_id
    left join questionnaire q on er.questionnaire_id = q.questionnaire_id
</sql>
```

3. **添加模糊搜索条件**：
```xml
<select id="selectEvaluationResultList" parameterType="EvaluationResult" resultMap="EvaluationResultResult">
    <include refid="selectEvaluationResultVo"/>
    <where>  
        <if test="studentId != null "> and er.student_id = #{studentId}</if>
        <!-- 新增：学生姓名模糊搜索 -->
        <if test="studentName != null and studentName != ''"> 
            and s.name like concat('%', #{studentName}, '%')
        </if>
        <if test="questionnaireId != null "> and er.questionnaire_id = #{questionnaireId}</if>
        <!-- 新增：问卷标题模糊搜索 -->
        <if test="questionnaireTitle != null and questionnaireTitle != ''"> 
            and q.title like concat('%', #{questionnaireTitle}, '%')
        </if>
        <!-- ... 其他条件 ... -->
    </where>
    order by er.create_time desc
</select>
```

## 📊 优化效果

### 优化前

| 搜索字段 | 用户友好度 | 实际使用频率 |
|---------|-----------|------------|
| 学生ID | ❌ 不友好 | 很少 |
| 问卷ID | ❌ 不友好 | 很少 |

**问题**：
- ❌ 用户通常不记得ID
- ❌ 需要先查询ID再搜索，操作繁琐
- ❌ 降低了搜索效率

### 优化后

| 搜索字段 | 用户友好度 | 实际使用频率 |
|---------|-----------|------------|
| 学生姓名 | ✅ 友好 | 很高 |
| 问卷标题 | ✅ 友好 | 很高 |

**优势**：
- ✅ 支持模糊搜索，输入部分姓名/标题即可
- ✅ 更符合用户使用习惯
- ✅ 提高搜索效率
- ✅ 数据展示更直观（表格中显示姓名和标题）

## ⚠️ 注意事项

1. **SQL注入防护**
   - 使用 MyBatis 的 `#{}` 而不是 `${}`
   - 已使用 `concat('%', #{studentName}, '%')` 安全方式

2. **空值处理**
   - 使用 LEFT JOIN 保证即使关联表无数据也能查询
   - 前端显示时处理 NULL 值

3. **性能考虑**
   - 模糊搜索（LIKE '%xxx%'）不能使用索引前缀
   - 如果性能不佳，考虑：
     - 添加全文索引
     - 使用 Elasticsearch
     - 限制搜索结果数量
