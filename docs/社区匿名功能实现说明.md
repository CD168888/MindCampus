```java
// 获取当前登录用户ID
Long userId = SecurityUtils.getUserId();

// 查询用户完整信息（包括头像）
SysUser sysUser = sysUserMapper.selectUserById(userId);

// 查询学生ID
LambdaQueryWrapper<Student> wrapper = new LambdaQueryWrapper<>();
wrapper.eq(Student::getUserId, userId);
Student student = studentInfoMapper.selectOne(wrapper);

// 设置发布用户信息（始终保存完整信息到数据库）
communityPost.setUserId(userId);
communityPost.setUserName(sysUser.getNickName());

```


### 2. Service层修改

#### CommunityPostServiceImpl.java

**修改点**:
- 添加 `StudentInfoMapper` 依赖
- 添加 `getStudentIdByUserId()` 私有方法
- 添加 `processAnonymousDisplay()` 公共方法（供Controller调用）
- 移除Service层的自动匿名处理，改为Controller层主动调用

```java
/**
 * 处理匿名显示逻辑（供APP端使用）
 * 如果is_anonymous='1'(匿名)，则隐藏真实用户信息
 */
public void processAnonymousDisplay(CommunityPost post) {
    if (post != null && "1".equals(post.getIsAnonymous())) {
        // 匿名帖子：使用默认头像和匿名用户名
        post.setUserName("匿名用户");
        post.setUserAvatar("/profile/avatar/default.png");
        // 注意：userId和studentId保留在数据库中
    }
}
```

#### CommunityCommentServiceImpl.java

**修改点**: 与帖子Service类似，添加匿名处理方法

## 📊 数据流程图

### 发布帖子流程

```
用户发布 → APP Controller
    ↓
1. 获取 userId (SecurityUtils)
2. 查询 SysUser (获取头像、昵称)
3. 查询 Student (获取 studentId)
4. 组装完整数据 + is_anonymous
    ↓
Service层 → Mapper → 数据库
    ↓
存储完整信息:
- user_id: 1
- student_id: 10
- user_name: "张三"
- user_avatar: "/profile/avatar/xxx.png"
- is_anonymous: "1" 或 "0"
```

### 查询帖子流程

```
APP端查询:
Controller → Service → Mapper → 数据库
    ↓
获取原始数据
    ↓
Controller 调用 processAnonymousDisplay()
    ↓
if (is_anonymous == '1'):
    userName = "匿名用户"
    userAvatar = "/profile/avatar/default.png"
    ↓
返回给前端

Web管理后台查询:
Controller → Service → Mapper → 数据库
    ↓
获取原始数据
    ↓
不调用 processAnonymousDisplay()
    ↓
直接返回完整信息给管理员
```
