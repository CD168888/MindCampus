## 为什么可能访问不到

### 1. 后端服务未重启

**原因：** 新创建的 Controller 需要重新编译和启动才能生效。

**解决方法：**
```bash
# 1. 停止当前运行的服务（在 IDEA 中点击停止按钮）

# 2. 重新编译项目
mvn clean compile

# 3. 重新启动服务（在 IDEA 中运行 McAdminApplication）
```

### 2. 数据库表结构未更新

**原因：** 新字段还没有添加到数据库中。

**解决方法：**
```sql
-- 执行以下 SQL 语句
ALTER TABLE `community_post`
ADD COLUMN `user_id` bigint(20) NULL COMMENT '用户ID' AFTER `post_id`,
ADD COLUMN `user_name` varchar(100) NULL COMMENT '用户名' AFTER `user_id`,
ADD COLUMN `user_avatar` varchar(500) NULL COMMENT '用户头像URL' AFTER `user_name`,
ADD COLUMN `title` varchar(200) NULL COMMENT '帖子标题' AFTER `user_avatar`,
ADD COLUMN `images` text NULL COMMENT '图片列表（逗号分隔）' AFTER `content`,
ADD COLUMN `like_count` bigint(20) DEFAULT 0 COMMENT '点赞数' AFTER `images`,
ADD COLUMN `comment_count` bigint(20) DEFAULT 0 COMMENT '评论数' AFTER `like_count`,
ADD COLUMN `view_count` bigint(20) DEFAULT 0 COMMENT '浏览数' AFTER `comment_count`;

-- 添加索引
CREATE INDEX `idx_user_id` ON `community_post`(`user_id`);
CREATE INDEX `idx_create_time` ON `community_post`(`create_time`);
CREATE INDEX `idx_status` ON `community_post`(`status`);
```

### 3. 编译错误

**原因：** 代码可能有编译错误导致服务启动失败。

**解决方法：**
- 在 IDEA 中查看 "Problems" 面板
- 检查控制台是否有错误信息
- 确保所有依赖都已正确导入

### 4. 端口冲突

**原因：** 8080 端口可能被占用。

**解决方法：**
```bash
# Windows
netstat -ano | findstr :8080
taskkill /PID <进程ID> /F

# 或者修改端口（application.yml）
server:
  port: 8081
```

## 常见错误及解决方法

### 错误 1：404 Not Found

**原因：**
- 服务未启动
- Controller 未加载
- 路径拼写错误

**解决：**
1. 确认服务已启动
2. 重启服务
3. 检查 URL 是否正确

### 错误 2：401 Unauthorized

**原因：**
- 未登录
- Token 过期
- Token 格式错误

**解决：**
1. 重新登录获取 token
2. 检查 Authorization header 格式：`Bearer YOUR_TOKEN`

### 错误 3：500 Internal Server Error

**原因：**
- 数据库连接失败
- SQL 执行错误
- 代码逻辑错误

**解决：**
1. 检查数据库连接
2. 查看后端日志
3. 确认数据库表结构已更新

### 错误 4：No static resource app/community/post/list

**原因：**
- Controller 未被 Spring 扫描到
- 包路径不正确
- 服务未重启

**解决：**
1. 确认 Controller 在正确的包下：`com.mc.community.controller`
2. 确认类上有 `@RestController` 注解
3. 重启服务

## 完整的重启流程

### 在 IDEA 中

1. **停止服务**
   - 点击红色方块停止按钮
   - 或按 Ctrl+F2

2. **清理编译**
   - 右键项目 → Maven → Clean
   - 或在 Terminal 执行：`mvn clean`

3. **重新编译**
   - 右键项目 → Maven → Compile
   - 或在 Terminal 执行：`mvn compile`

4. **启动服务**
   - 找到 `McAdminApplication.java`
   - 右键 → Run 'McAdminApplication'
   - 或按 Shift+F10

5. **等待启动完成**
   - 查看控制台，等待出现 "Started McAdminApplication"
   - 确认端口已监听：`Tomcat started on port(s): 8080`

### 使用命令行

```bash
# 1. 进入项目根目录
cd /path/to/MindCampus

# 2. 停止现有服务（如果在运行）
# 找到进程并杀死，或者在运行的终端按 Ctrl+C

# 3. 清理并编译
mvn clean package -DskipTests

# 4. 启动服务
java -jar mc-admin/target/mc-admin.jar

# 或者使用 Spring Boot Maven 插件
cd mc-admin
mvn spring-boot:run
```
