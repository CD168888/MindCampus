# 小程序上传图片配置说明

## 问题原因

小程序端上传图片失败的主要原因：

1. **baseUrl配置错误**：使用了 `http://localhost:8080`，小程序无法访问localhost
2. **未使用统一的上传工具**：直接使用 `uni.uploadFile` 而不是封装好的 `upload` 工具

## 解决方案

### 1. 修改 baseUrl 配置（必须）

打开 `mc-ui/MindCampus-App/config.js`，将 baseUrl 修改为您的真实IP地址或域名：

```javascript
module.exports = {
  // 错误的配置（小程序无法访问）
  // baseUrl: 'http://localhost:8080',
  
  // 正确的配置（使用真实IP地址）
  baseUrl: 'http://192.168.1.100:8080',  // 替换为您的实际IP地址
  
  // 或者使用域名
  // baseUrl: 'https://yourdomain.com',
  
  // 其他配置...
}
```

#### 如何获取本机IP地址：

**Windows系统：**
1. 打开命令提示符（cmd）
2. 输入 `ipconfig`
3. 找到 "IPv4 地址"，通常是 `192.168.x.x` 格式

**Mac/Linux系统：**
1. 打开终端
2. 输入 `ifconfig`
3. 找到 en0 或 eth0 下的 inet 地址

**注意事项：**
- 确保您的手机和电脑在同一个局域网内
- 如果使用微信开发者工具，需要在"详情 > 本地设置"中勾选"不校验合法域名"
- 正式发布时需要配置合法的HTTPS域名

### 2. 代码优化（已完成）

已将 `publish.vue` 中的上传方法改为使用统一的 `upload` 工具：

**修改前：**
```javascript
uni.uploadFile({
  url: config.baseUrl + '/common/upload',
  filePath: filePath,
  name: 'file',
  header: {
    'Authorization': 'Bearer ' + uni.getStorageSync('token')
  },
  // ...
})
```

**修改后：**
```javascript
import upload from '@/utils/upload'

upload({
  url: '/common/upload',
  filePath: filePath,
  name: 'file'
}).then(res => {
  return res.url
})
```

### 3. 后端配置检查

确保后端服务正常运行：

1. **启动后端服务**：运行 `bin/run.bat` 或在IDEA中启动项目
2. **检查端口**：确认服务运行在 8080 端口
3. **检查上传目录**：确保配置的上传路径存在
   - 路径配置在：`mc-admin/src/main/resources/application.yml`
   - 配置项：`ruoyi.profile: C:/Users/86173/Desktop/毕设项目/log/uploadPath`
4. **检查文件大小限制**：
   - 单个文件：10MB
   - 总大小：20MB

### 4. 测试步骤

1. 修改 `config.js` 中的 baseUrl 为真实IP
2. 重启小程序（编译运行）
3. 登录小程序
4. 进入发帖页面
5. 点击上传图片
6. 选择图片后查看上传结果

### 5. 常见问题

#### 问题1：仍然显示"无法访问该页面"
- 检查后端服务是否启动
- 检查IP地址是否正确
- 检查手机和电脑是否在同一网络
- 尝试在浏览器访问 `http://你的IP:8080/common/upload` 确认接口可访问

#### 问题2：上传超时
- 检查网络连接
- 图片文件是否过大（超过10MB）
- 增加 `utils/upload.js` 中的 timeout 值

#### 问题3：Token失效
- 重新登录小程序
- 检查token是否正确存储

#### 问题4：权限不足
- 检查用户是否已登录
- 检查用户权限配置

## 优化建议

### 1. 图片压缩

在上传前对图片进行压缩，减少上传时间和流量：

```javascript
chooseImage() {
  uni.chooseImage({
    count: 9 - this.form.images.length,
    sizeType: ['compressed'], // 添加压缩选项
    sourceType: ['album', 'camera'],
    success: (res) => {
      const tempFilePaths = res.tempFilePaths
      this.uploadImages(tempFilePaths)
    }
  })
}
```

### 2. 添加上传进度

修改 `utils/upload.js`，添加上传进度回调：

```javascript
const uploadTask = uni.uploadFile({
  // ...
  success: (res) => {
    // ...
  }
})

// 监听上传进度
uploadTask.onProgressUpdate((res) => {
  console.log('上传进度：' + res.progress)
})
```

### 3. 错误提示优化

在 `publish.vue` 中添加更详细的错误提示：

```javascript
.catch(err => {
  uni.hideLoading()
  let errorMsg = '上传失败'
  if (err.includes('timeout')) {
    errorMsg = '上传超时，请检查网络'
  } else if (err.includes('Network')) {
    errorMsg = '网络连接失败'
  }
  this.$modal.showToast(errorMsg)
})
```

## 总结

最关键的修改是：**将 `config.js` 中的 baseUrl 改为真实IP地址**。

修改后的完整流程：
1. 修改 baseUrl 为真实IP
2. 重启后端服务
3. 重新编译运行小程序
4. 测试上传功能

如果还有问题，请检查：
- 后端服务是否正常运行
- 网络连接是否正常
- Token是否有效
- 上传目录是否有写入权限
