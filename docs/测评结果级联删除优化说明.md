# 测评结果级联删除优化说明

## 📝 需求说明

删除测评结果（`evaluation_result`）时，应该同时删除关联的答题记录（`questionnaire_answer`），确保数据一致性。

## ✅ 实现方案

### 1. 级联删除逻辑

在删除测评结果时，先删除关联的答题记录，再删除测评结果本身。

**删除顺序**：
1. 删除 `questionnaire_answer` 表中 `result_id` 匹配的所有记录
2. 删除 `evaluation_result` 表中的记录

### 2. 事务保证

使用 `@Transactional` 注解确保删除操作的原子性：
- 如果删除答题记录失败，整个操作回滚
- 如果删除测评结果失败，整个操作回滚
- 保证数据一致性

## 🔧 修改内容

### 文件：`mc-project/src/main/java/com/mc/evaluation/service/impl/EvaluationResultServiceImpl.java`

#### 1. 添加依赖注入

```java
@Autowired
@Qualifier("questionnaireAnswerMapper")  // 明确指定使用 evaluation 包下的 Mapper
private QuestionnaireAnswerMapper questionnaireAnswerMapper;
```

**说明**：
- 注入 `evaluation` 包下的 `QuestionnaireAnswerMapper`
- 使用 `@Qualifier` 明确指定 Bean 名称，避免与 `questionnaire` 包下的同名 Mapper 冲突

#### 2. 修改单个删除方法

```java
@Override
@Transactional(rollbackFor = Exception.class)
public int deleteEvaluationResultByResultId(Long resultId) {
    // 先删除关联的答题记录
    questionnaireAnswerMapper.deleteByResultId(resultId);
    // 再删除测评结果
    return evaluationResultMapper.deleteEvaluationResultByResultId(resultId);
}
```

#### 3. 修改批量删除方法

```java
@Override
@Transactional(rollbackFor = Exception.class)
public int deleteEvaluationResultByResultIds(Long[] resultIds) {
    // 先删除关联的答题记录
    for (Long resultId : resultIds) {
        questionnaireAnswerMapper.deleteByResultId(resultId);
    }
    // 再删除测评结果
    return evaluationResultMapper.deleteEvaluationResultByResultIds(resultIds);
}
```

```
    ↓

    ↓

    ↓

```

## 🔍 关联关系

```
evaluation_result (测评结果表)
    ├─ result_id (主键)
    └─ student_id, questionnaire_id, ...

questionnaire_answer (答题记录表)
    ├─ answer_id (主键)
    ├─ result_id (外键，关联 evaluation_result.result_id)
    └─ question_id, user_answer, ...
```

**删除规则**：
- 一个测评结果（`result_id`）可以有多条答题记录
- 删除测评结果时，必须同时删除所有关联的答题记录

## ⚠️ 注意事项

### 1. Bean 名称冲突

由于存在两个同名的 `QuestionnaireAnswerMapper`：
- `com.mc.evaluation.mapper.QuestionnaireAnswerMapper` → Bean 名称：`questionnaireAnswerMapper`
- `com.mc.questionnaire.mapper.QuestionnaireAnswerMapper` → Bean 名称：`questionnaireAnswerRecordMapper`

使用 `@Qualifier("questionnaireAnswerMapper")` 明确指定使用 `evaluation` 包下的 Mapper。

### 2. 事务管理

- 使用 `@Transactional(rollbackFor = Exception.class)` 确保任何异常都会回滚
- 删除操作必须在同一个事务中完成
- 如果删除答题记录成功但删除测评结果失败，整个操作会回滚

### 3. 性能考虑

批量删除时，对每个 `resultId` 执行一次删除答题记录的操作。如果 `resultIds` 数量很大，可以考虑：
- 使用批量删除 SQL（如果 Mapper 支持）
- 或者优化为一次 SQL 删除所有关联记录

**当前实现**：
```java
for (Long resultId : resultIds) {
    questionnaireAnswerMapper.deleteByResultId(resultId);
}
```

**优化建议**（如果性能成为问题）：
```java
// 可以考虑在 Mapper 中添加批量删除方法
questionnaireAnswerMapper.deleteByResultIds(resultIds);
```
