## 📊 数据库表结构

```sql
CREATE TABLE questionnaire_answer (
    answer_id           BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '答题记录ID',
    result_id           BIGINT NOT NULL COMMENT '关联心理测评结果ID',
    questionnaire_id    BIGINT NOT NULL COMMENT '问卷ID',
    question_id         BIGINT NOT NULL COMMENT '题目ID',
    
    -- 冗余字段（用于历史追溯和快速统计）
    type                VARCHAR(20) NOT NULL COMMENT '题目类型（choice选择题/short_answer简答题）',
    content             VARCHAR(500) NOT NULL COMMENT '题干内容',
    options             JSON COMMENT '选择题选项（A/B/C/D…），简答题为空',
    standard_answer     VARCHAR(200) COMMENT '标准答案（仅选择题有效）',
    score               INT DEFAULT 0 COMMENT '分值',

    -- 用户作答部分
    user_answer         VARCHAR(500) COMMENT '用户作答内容',
    is_correct          TINYINT(1) DEFAULT NULL COMMENT '是否答对（1正确/0错误/2表示简答题 无）',
    obtain_score        INT DEFAULT 0 COMMENT '用户获得分数',

    create_by VARCHAR(64) DEFAULT '' COMMENT '创建者',
    create_time DATETIME COMMENT '创建时间',
    update_by VARCHAR(64) DEFAULT '' COMMENT '更新者',
    update_time DATETIME COMMENT '更新时间',
    remark VARCHAR(200) DEFAULT NULL COMMENT '备注'
) COMMENT='心理测评答题记录表';
```

## 🔍 功能特性

### 1. 搜索功能

支持以下搜索条件：
- **学生姓名**：模糊搜索
- **问卷标题**：模糊搜索
- **题目类型**：选择题/简答题
- **是否答对**：使用 `is_correct` 字典（1=正确，0=错误，2=无）

### 2. 列表展示

表格列包括：
- 答题记录ID
- 学生姓名（关联查询）
- 问卷标题（关联查询）
- 题目类型
- 题干内容
- 用户答案
- 标准答案
- 是否答对（字典显示）
- 分值
- 获得分数

### 3. CRUD 操作

- ✅ **查询**：支持分页、多条件搜索
- ✅ **新增**：添加新的答题记录
- ✅ **修改**：编辑现有答题记录
- ✅ **删除**：单个或批量删除
- ✅ **导出**：Excel 导出功能
- ✅ **查看**：详情查看对话框

### 4. 关联查询

通过 LEFT JOIN 实现：
```sql
questionnaire_answer qa
LEFT JOIN evaluation_result er ON qa.result_id = er.result_id
LEFT JOIN student_info s ON er.student_id = s.student_id
LEFT JOIN questionnaire q ON qa.questionnaire_id = q.questionnaire_id
```

## ⚠️ 注意事项

1. **数据库表结构**
   - 确保 `questionnaire_answer` 表已创建
   - 注意 `update_time` 字段后不要有多余的逗号

2. **字典配置**
   - 必须配置 `is_correct` 字典
   - 字典值：1（正确）、0（错误）、2（无）

3. **关联数据**
   - 确保 `evaluation_result`、`student_info`、`questionnaire` 表存在
   - 关联查询使用 LEFT JOIN，允许关联数据为空

4. **权限配置**
   - 执行菜单 SQL 后，需要为角色分配相应权限
   - 前端按钮使用 `v-hasPermi` 指令控制显示
